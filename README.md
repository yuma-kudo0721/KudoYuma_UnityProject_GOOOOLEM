# GOOOOLEM（東京ゲームショウ）

## 概要
本作『GOOOLEM』は、パズルでモンスターを召喚して戦う対戦型パズルゲームです。  
ピースとソウルを組み合わせてモンスターを召喚し、城を守りながら相手の城を攻めます。

シングルプレイでは、レベルが上がり続ける勇者パーティーをどこまで耐えられるかに挑戦します。  
マルチプレイでは2人対戦で互いにモンスターを召喚し合い、相手の城のHPを0にした方が勝利となります。

東京ゲームショウでの展示を想定して制作を開始し、  
テストプレイを通してキャラクター性能や挙動のバランス調整を行いました。

## 遊び方
1. タイトル画面でモードを選択  
2. ピースとソウルを一緒に消してモンスターを召喚（スペースキー選択開始、矢印キーピース範囲選択、Cキー完了） 
3. 大砲ゲージを貯めて大砲を発射 (ピースを選択開始していない場合にCキー)
4. 何レベルまで行けるかチャレンジ（ソロプレイ）
5. 相手の城のHPを０にしたら勝ち（マルチプレイ）

### 対応プラットフォーム
- PC

## 上手にプレイする方法
1. モンスターの能力や性質を把握する  
2. タイミングよく大砲を発射する  
3. 状況に応じたモンスターの召喚タイミングを見極める

## プログラム面（こだわり）
### ドラゴンデストロイヤー
- モンスター共通クラス・人間共通クラスを継承した職業専用キャラクターとして実装
- 攻撃処理に Coroutine を使用し、攻撃モーションを段階的に制御
- スプライト切り替えとダメージ判定を分離し、見た目と処理が一致する自然な攻撃表現を実現
- 特定の敵に対して強みを持たせ、「職業らしさ」をプログラム上で明確に表現

### 大砲
- 弾の生成・物理挙動だけでなく、発射時の反動演出まで含めて実装
- Rigidbody2D を用いたシンプルで安定した弾の挙動
- 発射時に砲身を後方へずらし、Lerp を用いて元の位置へ滑らかに復帰させる反動演出を実装
- 操作時の手応えや視覚的なフィードバックを重視した設計

### メタルスライム
- スライムはソウル1つで召喚される低コストモンスターとして設計
- 亜種モンスターを生成できない仕様上の制約に対し、一定確率でメタルスライムが出現する仕組みを実装
- 低コスト召喚でありながら、レア出現による期待感と戦略性を付加
- メタルスライムには通常モンスターとは異なる専用のダメージ処理を持たせ、戦況が変化しやすい設計

## ゲームバランス
- 友人とのテストプレイを通して、キャラクターの数値や挙動を何度も調整
- 大量にピースを消すだけの戦略にならないよう、
  低コストモンスターの召喚にも明確な意味が生まれるようバランスを設計

## まとめ
本作では、複数のシステムが絡み合うゲームにおいて、  
拡張性を意識した実装や、演出とロジックを分離する設計の重要性を学びました。

また、テストプレイを通じて数値や挙動を調整し、  
プレイヤー体験を改善していく開発プロセスを実践できたことは、  
今後のゲーム制作においても活かせる大きな経験になりました。
